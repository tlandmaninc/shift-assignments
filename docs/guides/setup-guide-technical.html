System Architecture

1 Prerequisites

Ensure you have the following accounts and tools before proceeding.

Accounts

  • GitHub — github.com (repository hosting)
  • Neon — neon.tech (serverless PostgreSQL)
  • Render — render.com (backend hosting)
  • Vercel — vercel.com (frontend hosting)
  • Gemini API Key — aistudio.google.com/app/apikey (AI chat)

Tools

  • Git — version control
  • Node.js 20+ — nodejs.org
  • Python 3.12+ — python.org
  • uv — astral.sh/uv (Python package manager)
  • Docker & Docker Compose — for local development

2 Clone & Local Dev

Get the codebase running locally with Docker Compose.

bash 
git clone https://github.com/YOUR_USERNAME/shift-assignments.git
cd shift-assignments
make setup    # copies .env.example files
make dev      # starts Docker Compose with hot-reload
Verify local setup: Frontend: http://localhost:3000 — Backend health: http://localhost:8000/health
WSL2 users: Run all commands from WSL, not Windows CMD/PowerShell. File watching relies on WATCHPACK_POLLING=true (already set in docker-compose.yaml).

3 Neon PostgreSQL Setup

Neon Set up a serverless PostgreSQL database in the Frankfurt region.

Create Project

  • Sign up at neon.tech
  • Create a new project: name shift-assignments, region aws-eu-central-1 (Frankfurt)
  • Copy the connection string from the Neon dashboard

Connection String Format

text 
postgresql://neondb_owner:PASSWORD@ep-XXXX.eu-central-1.aws.neon.tech/neondb?sslmode=require

Run Migration

bash 
DATABASE_URL="postgresql://neondb_owner:PASSWORD@ep-XXXX.eu-central-1.aws.neon.tech/neondb?sslmode=require" \
  uv run python backend/scripts/migrate_to_neon.py

Verify Connection

bash 
DATABASE_URL="postgresql://..." \
  uv run python -c "from backend.app.db import init_db; init_db(); print('OK')"
Note: The connection string contains your password. Never commit it to version control. Store it in environment variables only.

Deploy & Verify

bash 
curl https://YOUR-BACKEND.onrender.com/health
Free tier note: The service sleeps after 15 minutes of inactivity. The first request after sleep takes 30–60 seconds to spin up.

5 Vercel Frontend Deployment

Vercel Deploy the Next.js frontend with automatic builds on push.

Create Project

Go to Vercel Dashboard and select Add New → Project → Import GitHub repo.

SettingValue
Project Nameshift-assignments
FrameworkNext.js (auto-detected)
Root Directoryfrontend

Environment Variables

VariableValue
API_URL https://YOUR-BACKEND.onrender.com
Important: Use API_URL (not NEXT_PUBLIC_API_URL). This is a runtime server-side variable used by the Next.js API proxy route. It is not exposed to the browser.

Verify

After deployment, visit your Vercel URL and confirm the login page loads.

6 Connect & Verify

Wire everything together and run through the verification checklist.

Update Backend FRONTEND_URL

Go to your Render service's environment variables and update FRONTEND_URL to your actual Vercel URL (e.g., https://shift-assignments.vercel.app). Redeploy the service.

Google OAuth Setup (if using)

  • Go to Google Cloud Console → APIs & Services → Credentials → OAuth 2.0 Client
  • Add Authorized redirect URI:
    https://YOUR-APP.vercel.app/api/auth/google/callback

Verification Checklist

  • Backend health: curl https://YOUR-BACKEND.onrender.com/health
  • Frontend loads: visit your Vercel URL in a browser
  • Login works: complete the Google OAuth flow
  • Chat works: send a message, AI responds
  • API docs: https://YOUR-BACKEND.onrender.com/api/docs

8 Troubleshooting

Common issues and their fixes.

IssueCauseFix
502 Bad Gateway Backend sleeping (free tier) Wait 30–60 seconds for cold start, then retry
CORS error FRONTEND_URL mismatch Update env var on Render — must be exact URL with https://
Login fails OAuth redirect URI wrong Check Google Cloud Console, ensure redirect URI matches your Vercel URL
Chat error Gemini rate limit Wait 60 seconds (temporary cooldown), then try again
Build fails on Render Docker build error Check Render logs; ensure Dockerfile exists in backend/
Data not persisting DATABASE_URL not set Add Neon connection string to Render environment variables
